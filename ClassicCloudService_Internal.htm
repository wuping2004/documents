We'll retire the Cloud Services (classic) deployment model.

Migrate your cloud services to the new Cloud Services (extended support) before 30 June 2022

You're receiving this email because you use the Cloud Services (classic) deployment model in Azure Service Manager.

During the past many years, Azure has launched various PaaS and IaaS offerings based on Azure Resource Manager. Resource Manager is the new age deployment and management service for Azure with various capabilities such as regional resiliency, role-based access control (RBAC), tags, policy, and supports deployment templates. Several services such as Azure Kubernetes Service, Azure VMSS, Azure App Service, Azure Service Fabric and many other services are all based on Resource Manager. These new services replace legacy Azure Service Manager (ASM) based Cloud Services (classic), which will be retired on 30 June 2022 for all first party Microsoft Internal services.

You can also migrate your deployments to Cloud Services (extended support), which is a new Resource Manager based deployment model and provides feature parity with Cloud Services (classic). Cloud Services (extended support) provides two paths for you to migrate from ASM to Resource Manager:

  1. Deploy cloud services directly in Cloud Services (extended support) in Resource Manager and then delete the old Cloud Service (classic) in ASM.
  2. Use the in-place migration tool, which enables a seamless, platform orchestrated migration of existing Cloud Services (classic) deployments to Cloud Services (extended support).

Required action

All Cloud Services deployed using Cloud Services (classic) need to migrate before 30 June 2022. Please refer the Resources section to choose your migration path and plan to complete migration before this date.
